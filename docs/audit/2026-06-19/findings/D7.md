# D7 — Child-agent tool-call boundary

Trust contract acknowledged: the operator opts into
`claude --dangerously-skip-permissions`. The findings below are about
**bounding and observing** that decision, not retroactively forbidding it.

## F-D7-006
- severity: nice
- likelihood: medium
- title: The composed child prompt is not hashed into telemetry
- file: skills/bmad-story-automator/src/story_automator/core/tmux_runtime.py:319
- excerpt: |
    def _emit_tmux_spawned(session: str, project_root: str | None) -> None:
        ...
        _telemetry_emitter(project_root).emit(
            TmuxSessionSpawned(
                timestamp=iso_now(),
                run_id=current_run_id(project_root or get_project_root()),
                session_name=session,
                story_key=_story_key_from_session_name(session),
                pid=pid,
                pane_geometry=geometry,
            )
        )
- repro: |
    grep -n "prompt_sha\|prompt_hash" skills/bmad-story-automator/src/story_automator/core/telemetry_events.py ; echo "exit=$?"
    # exit=1: telemetry never records what the child was asked to do
- proposed_fix: Add `prompt_sha256: str` and `persona: str` to `TmuxSessionSpawned`. The parent already has the command before spawn_session is called; hashing it costs one `hashlib.sha256(command.encode()).hexdigest()`. Lets post-hoc operators answer "what did the orchestrator ask the child to do" without re-deriving the prompt from policy + workflow files.
- effort: S
- confidence: 4
