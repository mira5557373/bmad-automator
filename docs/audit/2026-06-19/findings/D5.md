## F-D5-001
- severity: should
- likelihood: low
- title: Unquoted RHS of ${var#prefix} treats TARGET_ROOT as glob pattern
- file: install.sh:71, install.sh:113
- excerpt: |
    echo "Backup: ${backup#$TARGET_ROOT/}"
    ...
    echo "Removed obsolete command shim: ${shim#$TARGET_ROOT/}"
- repro: |
    bash -c 'TARGET_ROOT="/foo*bar"; backup="/foo*bar/x.backup-1"; echo "[${backup#$TARGET_ROOT/}]"'
    # If a real directory at e.g. /tmp/proj* exists alongside /tmp/proj1, the prefix
    # in `${backup#…}` is matched as a shell glob, not as a literal string, so the
    # display name in the operator log can lose the leading path or — more dangerous
    # in CI — appear truncated/misleading after rename. shellcheck SC2295 flags both
    # occurrences.
- proposed_fix: Quote the RHS: `${backup#"$TARGET_ROOT"/}` and `${shim#"$TARGET_ROOT"/}`. Two-character change per line, no behaviour change for non-glob paths.
- effort: S
- confidence: 5

## F-D5-002
- severity: nice
- likelihood: low
- title: SC2155 — local+command-substitution masks date(1) exit status
- file: install.sh:69
- excerpt: |
    local backup="${path}.backup-$(date -u +%Y%m%dT%H%M%SZ)"
- repro: |
    # `local foo=$(cmd)` always returns success because `local` itself succeeds,
    # so `set -e` cannot catch a failure in `cmd`. For `date -u +…` this is purely
    # theoretical (the only realistic failure is OOM-fork or a broken libc).
- proposed_fix: Split: `local backup; backup="${path}.backup-$(date -u +%Y%m%dT%H%M%SZ)"`. One-line change, preserves behaviour under set -e and lets a date(1) failure surface.
- effort: S
- confidence: 4
