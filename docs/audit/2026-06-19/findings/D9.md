## F-D9-002
- severity: should
- likelihood: high
- title: 9 dataclass tests fail on declared min Python 3.11 — production code OK but CI signal poisoned
- file: tests/test_feature_tester.py:74; tests/test_gap_validator.py; tests/test_golden_trace_helpers.py; tests/test_spec_compliance.py
- excerpt: |
    # tests/test_feature_tester.py lines 68-74 (verbatim):
        def test_is_frozen_kw_only_dataclass(self) -> None:
            from story_automator.core.feature_tester import TestPlanEntry

            self.assertTrue(dataclasses.is_dataclass(TestPlanEntry))
            params = TestPlanEntry.__dataclass_params__  # type: ignore[attr-defined]
            self.assertTrue(params.frozen)
            self.assertTrue(params.kw_only)

    # On Python 3.11 this AttributeErrors because `_DataclassParams.kw_only`
    # was added in 3.12. The decorator `@dataclass(kw_only=True)` itself works
    # on 3.11 — only the introspection attribute is missing.
- repro: |
    /tmp/audit-venv-311/bin/python -m pytest tests/test_feature_tester.py::TestPlanEntryDataclassTests::test_is_frozen_kw_only_dataclass -v
    # -> AttributeError: '_DataclassParams' object has no attribute 'kw_only'
- proposed_fix: Either (a) bump `requires-python` to `>=3.12` and drop 3.11 from classifiers (matches what the tests actually exercise), or (b) gate the assertions on `sys.version_info >= (3, 12)` / use `inspect.signature(Cls).parameters` to confirm all params are KEYWORD_ONLY (works on 3.11). Option (a) is cleaner since the project already requires modern Python.
- effort: S
- confidence: 5
