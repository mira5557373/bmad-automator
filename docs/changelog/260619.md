# Changelog - 260619

## 260619-23:55:20 - [FULL] Resolve single-user-relevant findings from the 2026-06-19 audit

### Summary
Triaged the 71-finding production-readiness audit (`docs/audit/2026-06-19/`) against a single-operator-on-own-VPS threat model and current HEAD, then closed the 21 real, single-user-relevant findings. The audit's multi-tenant / untrusted-input hardening (D6 prompt-injection tripwires, D7 child sandboxing, audit-key `/proc` leakage) was deliberately skipped as out-of-model, as were findings already resolved in `aeecbe1` and pure test-coverage gaps.

### Added
- `doctor` preflight command: checks the Python version, the `filelock`/`psutil` dependencies, `tmux`/`claude`/`codex`/`git` on PATH, free disk, `BMAD_AUDIT_KEY` length, bundled-config validity, and the file-descriptor limit; JSON output with an optional `--human` summary (F-013).
- `telemetry-report --tail <N>`: a read-only live-debug view of the last N events that tolerates a corrupt tail line instead of aborting (F-052).
- Operator runbooks for a mid-run child-agent crash and audit-key rotation, plus an expanded error-code catalog in the CLI reference (F-014, F-025, F-026).
- Regression tests covering every fixed finding (`tests/test_audit_singleuser_fixes.py`, `tests/test_doctor.py`).

### Changed
- `build_calibration` correlates `StoryCompleted`/`StoryFailed` back to their `StoryStarted` `(model, complexity)` so the calibration table is no longer permanently empty on real ledgers, restoring M03 cost defaults and M08 drift baselines (F-001).
- `run_cmd` decodes child output leniently (`errors="replace"`) so a non-UTF-8 byte cannot raise `UnicodeDecodeError` and break the JSON error contract (F-067).
- Declared `filelock`/`psutil` in `pyproject` dependencies and added the Python 3.14 classifier (F-012, F-071).

### Fixed
- Recoverable-input crashes: non-dict `complexityOverrides`, non-dict config-json, and a non-object presets file no longer raise `AttributeError` (F-017, F-039); `is_stale` no longer raises on a `+00:00` heartbeat (F-015).
- Silently-wrong behavior: policy escalations read the real `message`/`severity` fields (F-016); `extract_last_action` returns the last action, not the first (F-037); the completion-marker regex no longer false-fires on log text such as "Tested for 3m 12s. Now starting next task." (F-036); `extract_json_block` handles trailing prose and nested objects (F-054); `filter_input_box` keeps markdown table rows (F-053); `update_simple_frontmatter` only rewrites the frontmatter region (F-034).
- `install.sh`: split the `local`+command-substitution so a `date` failure surfaces (SC2155), and quoted the `${var#prefix}` RHS (SC2295) (F-070, F-061).
- The `patch_env` test helper now isolates `AI_AGENT`/`AI_COMMAND`, fixing a non-deterministic `test_state_policy_metadata` failure under runners (e.g. Claude Code) that export `AI_AGENT`.
