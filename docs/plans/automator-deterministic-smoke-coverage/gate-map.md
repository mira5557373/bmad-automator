# Automator Deterministic Smoke Coverage Gate Map

| Gate | Owner/location | Local command | Env/reset/cache policy | CI status | Pass/fail signal | Failure diagnostic | Blocked/risk note |
| --- | --- | --- | --- | --- | --- | --- | --- |
| Version and metadata alignment | Phase 01 / `scripts/check-version-alignment.*` or equivalent | `npm run version:check` | No external reset. Reads repo metadata only. | Not CI-backed yet | Fails on mismatched package/plugin/module/Python/workflow versions | Command output should name mismatched files and values | Current workflow version appears stale. |
| Smoke input determinism | Phase 01 / `scripts/smoke_prep/config.py` and prep report | `npm run smoke:input-check` or folded into `npm run smoke:prepare -- --reset` report assertion | Must pin or record/assert BMAD Method installer input. | Not CI-backed yet | Fails when moving external input is unpinned or unrecorded | Prep report or assertion output | `bmad-method@next` is moving unless changed. |
| Package content assertion | Phase 02 / package assertion script | `npm run pack:assert` from `npm pack --dry-run --json` | No workspace reset. | Not CI-backed yet | Required files present, forbidden generated paths absent | JSON package file list diff | Should run in `verify`. |
| Packed tarball identity | Phase 02 / package assertion script and prep report | `npm run pack:assert` must include `npm pack --json --pack-destination <tmp>` identity assertions, or split as `npm run pack:identity` if needed | No workspace reset for pack; prep records generated tarball. | Not CI-backed yet | Name, version, filename, integrity or shasum, generated tarball path, and selected checksums match current package metadata | Pack JSON, tarball checksum output, and prep report | Prevents stale same-shape install from passing. |
| Prepared repo installed manifest | Phase 02 / `scripts/smoke_prep/automator.py` | `npm run smoke:prepare -- --reset` plus Phase 02 manifest assertion | Deletes and recreates `.smoke`; network needed unless cache present. | Not CI-backed yet | Installed tarball identity, installed version surfaces, installed-file checksums, and dependency skill entrypoints match current tarball | `.smoke` prep logs, tarball metadata, checksum diff, and manifest diff | External BMAD installer may drift. |
| Planning/create smoke | Existing / `scripts/run-smoke-automator.py` | `npm run smoke:run` | Requires prepared `.smoke/gunz`; resets story artifacts unless `--keep-artifacts`. | Not CI-backed yet | Prints `smoke run ok` and report path | `.smoke/AUTOMATED_SMOKE_REPORT.md` | Currently covers story `1.1` only. |
| Two-story dev-loop smoke | Existing / `scripts/run-smoke-dev-loop.py` | `npm run smoke:dev-loop` | Requires prepared `.smoke/gunz`; resets dev-loop artifacts unless `--keep-artifacts`. | Not CI-backed yet | Prints `dev-loop smoke ok` and report path | `.smoke/AUTOMATED_DEV_LOOP_SMOKE_REPORT.md` | Uses fixed parsed-dev fixture today. |
| Parser contract matrix | Phase 03 / `scripts/run-smoke-contracts.*` or tests | `npm run smoke:contracts` | Temp files and fake `claude`; no live LLM. | Not CI-backed yet | Success and fail-closed parse-output cases assert exit code plus JSON payload | Contract smoke JSON/report output | Must include empty/missing output, bad state flag, policy, contract, subprocess, no-json, decode, and schema failures. |
| Monitor terminal-state matrix | Phase 03 / `scripts/run-smoke-contracts.*` or tests | `npm run smoke:contracts` | Runner/temp state fixtures; no live LLM. | Not CI-backed yet | Every terminal state asserts `final_state`, `exit_reason`, `output_verified`, and diagnostics | Contract smoke JSON/report output | Monitor can exit 0 for failure states; assert JSON. |
| Runner lifecycle matrix | Phase 03 / `scripts/run-smoke-contracts.*` or tests | `npm run smoke:contracts` | `SA_TMUX_RUNTIME=runner`; temp dirs. | Not CI-backed yet | Success, crash, spawn_error, interrupted, and stuck mapping cases pass | Runner state/output files | Avoid real tmux when runner mode can substitute. |
| Build-cmd branch matrix | Phase 03 / `scripts/run-smoke-contracts.*` or tests | `npm run smoke:contracts` | Temp state/policy fixtures; no live LLM. | Not CI-backed yet | Codex flags, `AI_COMMAND`, Claude branch, model quoting, unknown step, invalid policy cases pass | Command string assertions and stderr | Default Codex branch alone is insufficient. |
| State policy verifier matrix | Phase 03 / `scripts/run-smoke-contracts.*` or tests | `npm run smoke:contracts` | Temp state/artifact fixtures. | Not CI-backed yet | State-update no-mutation failures, policy snapshot creation/failure, and success verifier edge cases pass | Contract smoke JSON/report output | Must verify policy snapshot creation and later pinned reads. |
| Marker/root resolution helper matrix | Phase 03 / `scripts/run-smoke-contracts.*` or tests | `npm run smoke:contracts` | Temp runtime-layout fixtures for `.agents`, `.codex`, and `.claude`; no live LLM. | Not CI-backed yet | Helper-resolved marker paths and runtime roots match each fixture and no hard-coded `.claude` path is accepted | Contract smoke JSON/report output and path assertions | Prevents marker drift across supported runtime roots. |
| Status/source-of-truth helper matrix | Phase 03 / `scripts/run-smoke-contracts.*` or tests | `npm run smoke:contracts` | Temp story-file and sprint-status fixtures. | Not CI-backed yet | Story-file status, sprint-status status, mismatch diagnostics, story-file fallback, and `sprint_status_not_updated` cases pass | Contract smoke JSON/report output and fixture file assertions | Prevents completion inference from marker absence or process exit alone. |
| Create startup guard smoke | Phase 04 / create-startup smoke runner | `npm run smoke:modes` or split `npm run smoke:create-startup` | Prefer temp BMAD-style fixtures; prepared `.smoke/gunz` only for realistic external flow. | Not CI-backed yet | Stop-hook states, existing-state detection, sprint-status present/missing, and init log checks pass | Startup smoke report and helper payloads | Must include pending-trust/changed/failure stop-hook cases where deterministic. |
| Resume/validate/edit smoke | Phase 04 / resume-validate-edit smoke runner | `npm run smoke:modes` | Seeded temp state files; no live LLM or prepared repo required for default gate. | Not CI-backed yet | Resume menu branches, marker, validate, edit branch helper contracts pass | Smoke report and seeded state diff | Interactive menu behavior must be tested through helper contracts, not prompts. |
| Mode state/artifact assertion matrix | Phase 04 / resume-validate-edit smoke runner | `npm run smoke:modes` | Seeded temp BMAD-style fixtures; no prepared repo needed for default gate. | Not CI-backed yet | Frontmatter, status, current story/step, agents/complexity files, policy snapshot path/hash, progress rows, action log deltas, reports, state docs, marker JSON, `.gitignore`, and selected artifacts match expectations | Smoke report, JSON payloads, and narrow file-content assertions | Prevents route-only smoke success. |
| Mode source-mismatch matrix | Phase 04 / resume-validate-edit smoke runner | `npm run smoke:modes` | Seeded story-file and sprint-status mismatch fixtures. | Not CI-backed yet | Story-file versus sprint-status mismatches are surfaced and do not get hidden by marker absence or exit status | Smoke report and mismatch diagnostics | Required source-of-truth behavior from Oracle review. |
| Finish-loop smoke | Phase 05 / finish-loop smoke runner | `npm run smoke:finish-loop` | Isolate commits inside `.smoke/gunz` or an explicit temp smoke repo; never commit host repo changes. | Not CI-backed yet | Automate/review/finalize/retro/wrapup statuses pass | Finish-loop smoke report and smoke repo `git log` | Commit isolation is critical. |
| Host commit isolation sentinel | Phase 05 / finish-loop smoke runner | `npm run smoke:finish-loop` | Record host HEAD/status before and after finish-loop; controlled commit only in smoke repo; refuse host repo target unless explicit unsafe override is supplied. | Not CI-backed yet | Host HEAD/status unchanged and smoke repo has expected commit | Host `git status`, host HEAD, smoke repo `git log` | Prevents accidental `commit-story --repo` targeting host repo. |
| Multi-epic retro fixture | Phase 05 / finish-loop smoke runner | `npm run smoke:finish-loop -- --scenario multi-epic`, or `npm run smoke:finish-loop` if that command always includes the multi-epic case | Seed two-epic story/status fixtures; no live LLM. | Not CI-backed yet | Epic 1 retro triggers before Epic 2 completion, Epic 1 retro failure is skipped/non-blocking, Epic 2 completes, separate `retrospectives.epic-*` entries exist, wrapup waits for all stories | Finish-loop smoke report and state document | Required by workflow multi-epic support; `smoke:deterministic-full` must run this scenario. |
| Full Python tests | Existing / `package.json` | `npm run test:python` | No external reset. | Not CI-backed yet | unittest reports OK | unittest output | None known. |
| CLI smoke | Existing / `package.json` | `npm run test:cli` | No external reset. | Not CI-backed yet | Both CLI help commands exit 0 | command stderr/stdout | None known. |
| Package dry run | Existing / `package.json` | `npm run pack:dry-run` | No external reset. | Not CI-backed yet | npm exits 0 | npm tarball file list | Needs assertions in Phase 02. |
| Shell smoke fixture | Existing / `scripts/smoke-test.sh` | `npm run test:smoke` | Uses repo fixture behavior. | Not CI-backed yet | script exits 0 | script output | Does not cover external prepared repo. |
| Final verify | Phase 06 / `package.json` | `npm run verify`: target `test:python && version:check && pack:assert && test:cli && smoke:contracts && smoke:modes && test:smoke` | No prepared-repo reset or network by default. | Not CI-backed yet | All selected fast deterministic gates pass | gate-map-driven command summary | `verify` does not yet include deterministic contract/mode smokes. |
| Full deterministic release smoke | Phase 06 / `package.json` | `npm run smoke:deterministic-full`: target `smoke:prepare -- --reset && smoke:run && smoke:dev-loop && smoke:finish-loop` | Prepared `.smoke/gunz`; reset/network/cache-heavy; explicit pre-release/nightly gate. | Not CI-backed yet | Prepared create, dev-loop, finish-loop, package identity, installed manifest, and host isolation pass | Combined smoke reports under `.smoke/` | Not suitable for default verify until CI cache and runtime budget are proven. |
